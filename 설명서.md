# 타임폴리오 차익거래 봇 설명서

## 1. 원리

### 1.1 가격 지연 차익거래 (Stale Price Arbitrage)

타임폴리오(한경 모의투자대회)의 종목 가격은 실제 시장 가격을 **9~19초 지연**하여 반영한다.
한국투자증권(KIS) Open API는 실시간 시세를 제공한다.

이 시간차를 이용하면:

```text
시점 T=0:  KIS = 221,500원  /  TF = 221,000원  (스프레드 +23bp)
           → TF에서 매수 (221,000원에 진입)

시점 T=15초: TF 가격 업데이트 → 221,500원
           → 수익 확정: +500원 (+23bp)
```

**핵심**: KIS가 "답지"이고, TF가 아직 답을 안 바꿨을 때 TF에서 사면 된다.

### 1.2 왜 손실이 0건인가

120초 모니터링 데이터 분석 결과:

| 종목 | 진입 횟수 | 승리 | 패배 | 보합 | 비고 |
|------|---------|------|------|------|------|
| 삼성전자 | 12 | 5 | **0** | 7 | 5틱 보유 시 |
| 대우건설 | 13 | 13 | **0** | 0 | 5틱 보유 시 **승률 100%** |
| ISC | 22 | 10 | **0** | 12 | 10틱 보유 시 |

패배가 없는 이유:
- KIS > TF일 때만 진입 → TF가 KIS를 따라잡으면 수익, 못 따라잡으면 보합
- TF가 KIS보다 더 떨어지는 경우가 관측되지 않음 (TF는 KIS를 추종하므로)
- "보합"은 TF가 아직 업데이트 안 한 것일 뿐, 더 기다리면 수렴

### 1.3 TF 가격 업데이트 주기 (실측)

| 종목 | 평균 업데이트 간격 | 최소 | 최대 |
|------|-----------------|------|------|
| 삼성전자 | 9초 | 3초 | 24초 |
| SK하이닉스 | 16초 | 3초 | 57초 |
| 대우건설 | 11초 | 3초 | 21초 |
| ISC | 19초 | 3초 | 51초 |
| 대한유화 | 16초 | 2초 | 44초 |

→ 유동성 높은 종목(삼성전자)이 더 자주 업데이트

```text
                                   │                  │
┌─────────────┐     REST API      │  1초 간격 감시    │
│ Timefolio   │◄──── 가격 조회 ───│  스프레드 계산    │
│  (대회 서버)  │     주문 제출 ───►│  자동 매수 주문   │
└─────────────┘                    └──────────────────┘
```

### 2.1 발견된 타임폴리오 내부 API

브라우저 네트워크 캡처 + JS 소스 리버스 엔지니어링으로 확인:

| 엔드포인트 | 메서드 | 용도 |
|-----------|--------|------|
| `/api/Auth/Login` | POST | JWT 토큰 발급 |
| `/api/Contest/PfList?ctstId=86&d=YYYY-MM-DD` | GET | 전체 랭킹 (2089명) |
| `/api/Contest/TopRankDetail?pfId={id}&d=YYYY-MM-DD` | GET | 유저 포트폴리오 상세 |
| `/api/Util/SearchProducts?term={검색어}` | GET | 종목 검색 |
| `/api/Portfolio/AddOrder` | POST | **주문 제출** |
| `/api/Portfolio/Orders?pfId={id}&d=YYYY-MM-DD` | GET | 주문 목록 |
| `wss://hankyung.timefolio.net/hubs/main` | WSS | 실시간 데이터 (SignalR) |

### 2.2 주문 API 본문 (JS 소스에서 확인)

```json
{
  "d": "2026-04-16",
  "pfId": 19685,
  "prodId": "A005930",
  "ls": "L",              // L=Long, S=Short
  "ex": "E",              // E=Entry(매수), X=Exit(매도)
  "wei": 1.0,             // 비중 %
  "exitAll": false,
  "limitPrc": null,        // 지정가 (상대호가 시 null)
  "limitIdx": 5,           // 상대호가 틱수 (1~10)
  "stopPrc": null,         // STOP가격 (미사용 시 null)
  "hm0": "09:30",          // 주문 시작 시간
  "hm1": null              // 종료 시간 (즉시실행 시 null)
}
```

## 4. 사용법

### 4.1 설치

```bash
pip install -r requirements.txt
```

### 4.2 .env 설정

```env
# 타임폴리오 계정
TIMEFOLIO_ID=your@email.com
TIMEFOLIO_PW=yourpassword

# KIS API (한국투자증권 Open API)
KIS_APP_KEY=your_app_key
KIS_APP_SECRET=your_app_secret
KIS_ACCOUNT=12345678

# 차익거래 설정
ARB_DRY_RUN=true       # true=주문 안 넣음, false=실제 주문
ARB_TOP_N=5            # 상위 N명 포트폴리오 모니터링
```

### 4.3 커맨드

```bash
# 차익거래 루프 봇 (핵심)
python run.py arb-loop-dry    # DRY_RUN: 주문 안 넣고 시뮬레이션
python run.py arb-loop        # LIVE: 실제 주문 (.env ARB_DRY_RUN 설정 따름)

# 1회성 차익거래 스캔
python run.py arb-dry         # DRY_RUN
python run.py arb             # LIVE

# 스프레드 데이터 수집 (120초)
python spread_monitor.py

# 기존 기능
python run.py scrape          # 포트폴리오 스크래핑
python run.py analyze         # 시그널 분석
python run.py all             # 스크래핑 + 분석
```

### 4.4 루프 봇 환경변수 (선택)

```bash
ENTRY_BPS=10            # 진입 임계값 (bp). 기본 10
ORDER_WEIGHT=1.0        # 주문당 비중%. 기본 1.0
MAX_POSITIONS=10        # 최대 동시 포지션. 기본 10
SCAN_INTERVAL=1.5       # 스캔 간격(초). 기본 1.0
MAX_TOTAL_WEIGHT=50.0   # 최대 총 비중%. 기본 50.0
```

## 5. 동작 흐름 (arb_loop.py)

```text
[초기화]
  1. 타임폴리오 REST API 로그인 → JWT 토큰
  2. KIS API OAuth 토큰 발급
  3. 상위 N명 포트폴리오에서 모니터링 종목 추출

[루프] (1초 간격, 장 종료까지)
  4. TF 가격 조회 (병렬 HTTP)  ─┐
  5. KIS 가격 조회 (병렬 HTTP) ─┘ 동시 실행

  6. 스프레드 계산:
     spread_bps = (KIS - TF) / TF × 10000

  7. 진입 판단:
     IF spread_bps > ENTRY_BPS (기본 10bp)
     AND 해당 종목 미보유
     AND 포지션 < MAX_POSITIONS
     → POST /api/Portfolio/AddOrder (즉시 매수)

  8. 수렴 확인:
     IF TF_현재가 >= 진입시_KIS가격
     → 수익 확정, 포지션 제거

  9. 로그 기록 → database/arb_trades_*.csv

[종료] Ctrl+C 또는 장 마감
  10. 요약 출력 + 텔레그램 알림
```

## 7. 리스크 및 주의사항

### 7.1 기술적 리스크

| 리스크 | 대응 |
|--------|------|
| KIS API 속도 제한 (초당 20건) | ThreadPool max_workers=10으로 제한 |
| 타임폴리오 API 변경 | probe_api.py로 재검증 |
| JWT 토큰 만료 | 자동 재로그인 필요 (미구현, 추후 추가) |
| 네트워크 에러 | try/except로 스캔 건너뛰고 계속 |

### 7.2 전략적 리스크

| 리스크 | 설명 |
|--------|------|
| 스프레드 소멸 | TF가 실시간 업데이트로 전환하면 전략 무효화 |
| 대회 약관 위반 가능성 | 자동매매 금지 조항 확인 필요 |
| 상대호가 슬리피지 | limitIdx=5로 5틱 위에 주문 → 스프레드가 5틱 이하면 손실 가능 |
| 유동성 리스크 | 소형주는 호가 스프레드가 커서 실체결 가격 불리 |

### 7.3 안전 장치

- `ARB_DRY_RUN=true` 기본값: 실수로 주문 안 나감
- `MAX_POSITIONS=10`: 과도한 포지션 방지
- `MAX_TOTAL_WEIGHT=50%`: 전체 포트폴리오 비중 제한
- 장외시간 자동 DRY_RUN 전환
- 모든 거래 CSV 로그 기록
